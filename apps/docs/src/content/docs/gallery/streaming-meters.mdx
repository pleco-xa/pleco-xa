title: Streaming meters
description: Worker-safe RMS and spectral-flux meters that emit the same values whether you feed them one buffer or a stream of random-sized chunks — bit-for-bit identical to the offline analysis.
---

import DemoEmbed from '../../../components/DemoEmbed.astro'

Real-time meters only earn trust if the answer doesn't depend on how the audio
arrives. This demo pushes a 0.5-amplitude 440 Hz sine into `createRmsMeter`
twice — once as a single buffer, once as seeded-random 37-to-1999-sample chunks —
and the two runs come out *bitwise identical*, every emitted RMS within 1% of
the analytic `0.5 / √2`. A companion `createFluxAnalyzer` watches an amplitude
step streamed in chunk by chunk and localizes it to within one frame, with the
very first frame reporting flux of exactly `0`.

<DemoEmbed src="streaming-meters.html" title="Chunk-size-invariant streaming meters" height={640} />

The meters are `createRmsMeter` and `createFluxAnalyzer`: `push()` returns only
the newly completed frames and `read()` is non-consuming, so the same analyzer
object drives a live UI and still totals up to the offline result. Because the
math carries its own frame-boundary state, chunk size is irrelevant — the same
code runs in the main thread, an AudioWorklet, or a Web Worker.
